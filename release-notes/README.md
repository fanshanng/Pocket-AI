# Release Notes

This folder keeps per-version development and test records.

Use this naming convention for future versions:

```text
RELEASE_NOTES_vX.Y.Z.md
```

Each version note should include:

- Version number
- Change summary
- Modified files
- New files
- Test results
- Release APK SHA256, if a release APK was built
- Git sync / commit / tag / push status

GitHub Release page drafts can also live here with this naming convention:

```text
github-release-vX.Y.Z.md
```
